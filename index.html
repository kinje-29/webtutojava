Pengenalan Java

Java adalah bahasa pemrograman yang fenomenal. Sebagaian ahli mengkalim bahwa kelebihan-kelebihan pada bahasa pemrograman sebelumnya telah diadopsi oleh java. Dalam hal kepopuleran, menurut rilis TIOBE Index (www.tiobe.com) tiap bulan, java berada di peringkat dua besar (bergantian dengan C yang menjadi peringkat pertama) dalam tida tahun terakhir.

Java bisa digunakan pada berbagai keperluan. Pemrograman berbasis teks pada Clipper atau FoxPro for DOS yang bisa kita temui di kasir-kasir minimarket bisa dilakukan dengan java. Pemrograman berbasis GUI Desktop seperti yang dikenal pada Visual Basic, Visual FoxPro, atau Delphi juga bisa dilakukan dengan java. Pemrograman untuk mikro seperti ponsel atau tablet PC mudah dilakukan dengan java. Tidak ketinggalan, pemrogram web bisa menggunakan java Server Pages(JSP) atau Servlet. Anda mau membuat perangkat lunak untuk analisis, pengolah citra digital, game, atau utilitas. Java bisa menanganinya dengan sangat baik.

java Memiliki tiga edisi berbagai keperluan di atas, yaitu

  • Java 2 Standard Edition (J2SE) untuk pemrograman pada komputer Desktop.
  • Java 2 Microdevice Edition (J2ME) untuk pemrograman pada peranti mikro seperti ponsel.
  • Java 2 Enterprise Edition (J2EE) untuk membangun multi-tier enterprise seperti jaba bean dan Servlet.

Semuah edisi java tersebut bisa didapatkan secara geratis dan legal. Anda hanya perlu mengunduhnya dari situs resmi Java.com atau oracle.com. Java adalah sumber terbuka (Open Source) sehingga program berikut kode yang digunakan untuk pengembangan perangkat lunak bebas diunduh dan bebas di modifikasi. Kita bisa memakai Java di windows, linux, ataupun Sun Solaris. Digunakan di Macintosh pun tidak masalah. Dengan kata lain, java bisa digunakan pada berbagai platfrom (multipltfrom), sesuai dengan slogannya : "tulis program sekali, jalakan di mana saja" (Write once run anywhere).

Java juga mendukung fitur-fitur pengembangan perangkat lunak terbaru seperti object orienred programming (OOP). Java memiliki tingkat keamanan yang lebih baik dari kebanyakan bahasa pemrograman lain. Komunitas pengembangan java pun banyak. Bahkan untuk belajar java kita bisa jumpai di Internet. Hal-hal tersebut menyebabkan java terus berkembang.

Sedikit kelebihan java
  • Mengakomodasi kelebihan-kelebihan bahasa pemrograman sebelumnya.
  • Popularitas tinggi, hingga saat ini, berdasarkan TIOBE Index
  • Multiguna (bisa dipakai untuk berbagai keperluan): pemrogram berbasis teks, GUI (Desktop), pemrogram mobile, utilitas, grafis, game, web, DLL
  • Grafis dan sumber terbuka
  • Multipltfrom
  • Mengakomodasi fitur-fitur pengembangan perangkat lunak terbaru seperti OOP
  • Didukung Komunitas, referensi, dan pustaka (library) yang berlimpah.

Dalam bahsa pemrograman pasti ada kelebihan dan kekurangan masing-masing, demikian juga dengan java. Beberapa kekurangan java berikut bisa jadi bahan referensi bagi anda yang ingin belajar dalam bahasa pemrogram java, dibawah ini kekurang dalam bahasa pemrogram java.

  • Lama waktu pengeksekusian program relatif lebih tinggi dibandingkan dengan C. Hal ini disebabkan penerjemahan kode dilakukan lebih dari satu tingkat dengan adanya java Virtual Machine. Akan tetapi dengan adanya teknologi baru Just In Time (JIT) bisa mngurangi atau menambal celah tersebut.
  • Membutuhkan memori yang relatif lebih besar. Konsumen memori pada komputer cukup besar.
  • Mudah didekompilasi. Programer sulit menyembunyikan kode sumber yang telah dihasilkannya.
  • Pengodeannya relatif lebih sulit.

Dasar Dasar Java dengan menggunakan IDE Netbeans

Netbeans adalah merupakan IDE yang ditujukan untuk memudahkan pemrograman java. Dalam Netbeans, pemrograman dilakukan berbasis visual dan event driver. Persis seperti IDE lain, misalnya Borland Delphi dan Micrsoft Visual Studio.

Untuk membuat dialog atau user-interface, kita tidak perlu membuat teks program secara manual baris per baris, tetapi cukup klik pada component pallete. Teks program akan dihasilkan secara otomatis. Netbeans mencakup compiler atau builder, dan debugger internal. Hal ini sangat memudahkan proses paska perancangan program. Porses deployment dan atau tanpa tes dapat juga dilakukan dari dalam Netbeans.

Cara menggunakan Netbeans untuk menjalankan java.

  • Pastikan dulu anda sudah menginsal java di computer anda
  • Kemudian download juga Netbeans Link download ada di atas
  • Install Netbeans di komputer anda.
  • Setalah kedua software siap, jalankan Netbeans

Langkah-Langkah Pratikum

  • Jalankan Netbeans pada komputer atau Laptop kalian
  • Jika sudah terbuka Netbeans sekarang pilih (Menu Fille)
  • Pilih sub menu New Project
  • Pilih General, pada pilihan Categories
  • Pilih Java Aplication pada pilihan Project
  • Klik Next
  • Setelah mengklik next akan ditampilkan menu untuk membuat project
  • Dibagian Project Name Kalian bisa menuliskan nama Project kalian masing masing tanpa ada tanda sepasi
  • Pada bagian Project Location kalian bisa memilih lokasi penyimpanan Project kalian, atau kalian bisa membiarkan nya menjadi difauled
  • Pada bagian Creat Main Class anda bisa Centang maka akan otomatis membuatkan kelas main atau anda bisa mengganti nama main class nya
  • Lalu setelah semuah sudah siap Klik Finis maka akan ditampilkan sebuah Project yang kalian buat
  • Kemudian pada bagian Sourec Package kalian klik nama Class yang kalian buat sebelumnya atau anda bisa membuat class baru dengan kalik kanan pada nama project kalian lalu pilih (New Java Class).

Java GUI (Graphical User Interface) adalah langkah instruksional yang sangat baik untuk mempelajari pemrograman berorientasi objek.

Rancangan API pada pemrograman GUI Java adalah contoh yang sangat relevan tentang bagaimana Anda bisa menerapkan prinsi-prinsip berorientasi objek.


Perbedaan Swing dan AWT

Swing GUI memiliki komponen-komponen yang stabil dan lebih fleksibel untuk menggantikan AWT GUI. Anda juga akan belajar membuat GUI sederhana dan yang terpenting bisa mendemonstrasikan dan memahami Pemrograman Berorientasi Objek dalam menampilkan komponen-komponen GUI, seperti:

  • JButton
  • JLabel
  • JTextField
  • JRadioButton
  • JComboBox

Setelah Anda melihat class komponen GUI itu, mungkin Anda bertanya-tanya kenapa awalannya semuanya memiliki huruf J? Misalnya JButton, kenapa tidak dinamai hanya sebagai Button saja, bisa saja kan? Itu karena sebenarnya sudah ada class Button di package java.awt.

Jadi class GUI ada di library yang dikenal sebagai Abstract Windows Toolkit (AWT) saat awal Java diperkenalkan. Tapi AWT itu tidak mampu untuk mengembangkan proyek GUI dengan antarmuka grafis yang kompleks atau komprehensif. Selain itu AWT juga rentan dengan bug pada platforms yang sepesifik.

Oleh sebab itulah komponen Swing hadir dengan keserbagunaannya, fleksibilitas, dan library yang lebih kuat untuk menggantikan komponen AWT tersebut. Komponen Swing di-paint secara langsung dengan kode Java namun komponen dari subclass Window dan Panel harus di gambar dengan GUI native pada platform spesifik.

Komponen Swing menggunakan lebih sedikit sumber daya GUI native dan tidak bergantung pada platform target. Oleh sebab itu komponen Swing itu disebut dengan lightweight component. Sedangkan komponen AWT disebut sebagai heavyweight component.

Class dari komponen Swing GUI memiliki awalan J, jadi akan mudah membedakannya dari class AWT. Oracle masih mendukung Swing dan AWT dan mereka menyatakan itu masih merupakan teknologi inti mereka dalam semua perilisan dan jadwal support Java SE.
